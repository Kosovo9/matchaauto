## PARTE 2: ARQUITECTURA Y SEGURIDAD 1000% EDGE

Para lograr la velocidad cuántica (Quantum Speed) y la seguridad nivel Dios, la arquitectura debe ser **Serverless First** y operar completamente en el **Edge** (borde de la red). Esto elimina la latencia geográfica y permite un escalado infinito.

### 2.1. Stack Tecnológico: El Tridente de la Velocidad

| Componente | Tecnología Seleccionada | Razón de la Elección (1000% Optimizado) |
| :--- | :--- | :--- |
| **Frontend** | **Next.js 15** (RSC) + **TailwindCSS 4.0** | Velocidad de carga instantánea, componentes de servidor para cero JavaScript en el cliente, UX macro. |
| **Compute/API** | **Cloudflare Workers** + **Hono.js** | Latencia global <50ms (Edge Computing). Escalado infinito y costo marginal. |
| **Autenticación** | **Clerk** | Solución de autenticación 100% segura, lista para la escala, con soporte nativo para **Apple, Google, Email/Password** y verificación por correo. |
| **Base de Datos** | **Supabase PostgreSQL** (Sharded) + **Cloudflare D1** | PostgreSQL para datos relacionales críticos. D1 (SQLite en el Edge) para datos de baja latencia y caché. |
| **Búsqueda** | **Meilisearch** (o Typesense) | Motor de búsqueda open source, tolerante a errores tipográficos, con geo-search instantáneo. |
| **Imágenes (Storage)** | **Cloudflare R2** + **Cloudflare Images** | **Costo mínimo** (cero egreso de datos), almacenamiento S3-compatible, optimización y redimensionamiento automático de imágenes en el Edge. |
| **Videos (Storage)** | **Cloudflare Stream** | **Entrega de video de alta calidad** (HLS/DASH) con transcodificación automática y DRM opcional, esencial para los planes VP. |
| **CDN/Seguridad** | **Cloudflare** (WAF, Bot Management, DDoS) | La red más grande del mundo, esencial para la seguridad nivel Dios y la velocidad global. |

### 2.2. Estrategia de Almacenamiento Global de Datos

#### A. Base de Datos de Ubicaciones Globales (Macro, Sin Lags)

Para soportar la expansión global sin latencia, implementaremos una base de datos geográfica de dos niveles:

1.  **Nivel 1 (Edge - Cloudflare D1):** Almacenará la **lista completa de países, estados/provincias y las 50,000 ciudades más pobladas del mundo** con sus coordenadas (lat/lon). Esto permite que las consultas de autocompletado y geo-búsqueda se resuelvan en el nodo más cercano al usuario (Edge), garantizando una velocidad cuántica.
2.  **Nivel 2 (Primary - Supabase PostgreSQL):** Utilizaremos la extensión **PostGIS** para manejar las consultas geográficas complejas (ej. "Listings dentro de un radio de 50km"). El sharding por región garantizará que la base de datos principal no se sature.

#### B. Almacenamiento de Medios (Imágenes y Videos)

La clave para el bajo costo y la alta velocidad es **Cloudflare R2** y **Cloudflare Stream**.

*   **Imágenes:** Todas las fotos de los listings se almacenarán en **R2**. Cloudflare Images se encargará de la compresión (WebP/AVIF) y el redimensionamiento al instante, reduciendo el tamaño de la página y el costo de almacenamiento.
*   **Videos (Planes VP):** Los videos se subirán a **Cloudflare Stream**. Esto no solo los almacena, sino que los transcodifica automáticamente a múltiples resoluciones y los sirve a través de su CDN, asegurando que un usuario en México o Japón vea el video sin *buffering* y a la máxima velocidad posible.

### 2.3. Seguridad Nivel Dios (NASA + FBI + CIA + Bank of America)

La seguridad no es un *feature*, es la base de la plataforma.

| Amenaza | Estrategia de Mitigación (1000% Blindada) |
| :--- | :--- |
| **Anti-DDoS** | **Cloudflare Spectrum/Magic Transit** + **WAF** (Web Application Firewall) en el Edge. Mitigación de ataques de capa 3, 4 y 7. |
| **Anti-Scraping (500%)** | **Cloudflare Bot Management Enterprise** (ML-driven). Detección de bots por comportamiento, *fingerprinting* de dispositivos, *rate limiting* estricto por endpoint. |
| **Anti-Cloning/Screenshot** | **Watermarking Dinámico** en imágenes (marca de agua invisible con ID de usuario y timestamp) + **Tokenización de Imágenes** (URLs que expiran en minutos). Anti-screenshot es imposible de bloquear al 100% en web, pero el *watermarking* dinámico desincentiva el robo de contenido. |
| **Anti-Spam/Fraude** | **AI/ML Moderation** (Hugging Face) en el pipeline de publicación. Detección de texto y patrones de fraude antes de que el listing se active. |
| **Anti-Hacking** | **Zero Trust Architecture** (Cloudflare Access). **RLS (Row Level Security)** en Supabase. **Input Validation** estricta con Zod en el Edge. |
| **Autenticación** | **Clerk** (JWTs seguros, 2FA obligatorio para vendedores, gestión de sesiones centralizada). |

### 2.4. Escalabilidad Global (Miles de Usuarios Simultáneos)

La arquitectura Serverless + Edge garantiza que la plataforma puede manejar **miles de usuarios simultáneos globalmente sin lags**.

*   **Latencia Cero:** Al operar en los 330+ PoPs (Puntos de Presencia) de Cloudflare, el contenido y la lógica de la aplicación están a milisegundos de cualquier usuario en el mundo.
*   **Auto-Escalado:** Cloudflare Workers escala automáticamente a millones de peticiones por segundo sin necesidad de configuración manual.
*   **Conexiones de Base de Datos:** Usaremos **PgBouncer** (o un equivalente serverless) para manejar el *connection pooling* y evitar que la base de datos principal se sature con miles de conexiones simultáneas.

Un porcentaje del ingreso publicitario se transferirá automáticamente a esta cuenta, asegurando que el capital para pagar a los afiliados esté siempre disponible y segregado de los fondos operativos.
